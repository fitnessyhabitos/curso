# 📖 Módulo 0 – Bienvenida y Preparación

## ¿Qué es la programación?

¡Hola! Bienvenido o bienvenida a este curso de programación.

Imagina que tienes un robot que hace exactamente lo que tú le dices, pero el robot es un poco "tonto": **solo entiende instrucciones muy precisas y en un idioma especial**. Ese idioma especial se llama **lenguaje de programación**.

Cuando escribimos instrucciones para el ordenador, a eso se le llama **programar** o **escribir código**.

```
Tú piensas una idea  →  La escribes en código  →  El ordenador la ejecuta  →  ¡Magia!
```

## 🐢 ¿Qué es Logo?

**Logo** es un lenguaje de programación inventado en 1967 por un señor muy listo llamado **Seymour Papert**. Fue diseñado especialmente **para que los niños aprendan a programar**.

La idea es sencilla: hay una **tortuga** en la pantalla y tú le das órdenes. La tortuga obedece y, mientras se mueve, va dejando un rastro dibujado.

```
¡Muévete hacia adelante!   →   FORWARD 100
¡Gira a la derecha!        →   RIGHT 90
¡Dibuja un cuadrado!       →   (combinando varios comandos)
```

Lo genial de Logo es que puedes ver **inmediatamente** en la pantalla lo que hace tu código. ¡Es como darle instrucciones a un robot que dibuja!

## 🐍 ¿Qué es Python?

**Python** es uno de los lenguajes de programación más populares del mundo. Se llama Python porque a su creador, Guido van Rossum, le gustaban los Monty Python (un grupo de comedia inglés).

Python es especial porque:
- Es **fácil de leer** – parece casi inglés normal
- Es **muy potente** – Google, YouTube y NASA lo usan
- Es **gratis** – cualquiera puede descargarlo
- Tiene **miles de "librerías"** – herramientas para hacer cualquier cosa

Después de aprender Logo, pasaremos a Python para crear programas de verdad.

## 💻 ¿Qué necesito instalar?

### Para Logo:
Vamos a usar **FMSLogo** (o una alternativa online).

**Opción 1 – FMSLogo (Windows):**
1. Ve a: `https://fmslogo.sourceforge.net/`
2. Descarga la última versión
3. Instala el programa normalmente
4. ¡Ábrelo y verás la tortuga!

**Opción 2 – Online sin instalar nada:**
- Visita: `https://papert.me/` o `https://www.calormen.com/jslogo/`
- Funciona directamente en el navegador

### Para Python:
Vamos a usar **Python 3 + Thonny** (el mejor editor para aprender).

**Paso 1 – Instalar Python:**
1. Ve a: `https://www.python.org/downloads/`
2. Descarga la versión más reciente (Python 3.x.x)
3. **¡IMPORTANTE!** Marca la casilla "Add Python to PATH" antes de instalar
4. Haz clic en "Install Now"

**Paso 2 – Instalar Thonny:**
1. Ve a: `https://thonny.org/`
2. Descarga e instala para Windows
3. Abre Thonny – ¡ya estás listo!

**Opción alternativa online:**
- `https://repl.it` – No necesitas instalar nada, funciona en el navegador

### Reglas de oro del buen programador:

1. **Lee primero, escribe después** – Entiende la teoría antes de copiar código
2. **Escribe a mano si puedes** – Ayuda a memorizar los comandos
3. **Experimenta** – Cambia un número y ve qué pasa. ¡Así es como se aprende!
4. **No copies y pegues** – Escribe el código tú mismo, aunque sea lento
5. **Los errores son amigos** – Un error no significa que eres malo, ¡significa que estás aprendiendo!
6. **Guarda tu trabajo** – Guarda los archivos con nombres descriptivos

## 🧠 Conceptos básicos que debes conocer

### ¿Qué es un algoritmo?

Un **algoritmo** es una lista de pasos ordenados para resolver un problema.

Por ejemplo, el algoritmo para hacer un sándwich sería:
```
1. Coge dos rebanadas de pan
2. Unta mantequilla en una rebanada
3. Pon el jamón encima
4. Coloca la otra rebanada encima
5. ¡Listo! Come el sándwich
```

¡Cuando programas, estás escribiendo algoritmos para el ordenador!

### ¿Qué es un error (bug)?

Un **bug** es un error en el código. Cuando hay un bug, el programa no funciona como esperamos.

La primera "persona" que encontró un bug de ordenador fue la científica **Grace Hopper** en 1947. Literalmente encontró un insecto (una polilla) dentro de un ordenador que lo hacía fallar. ¡Por eso los errores se llaman "bugs" (insectos en inglés)!

Cuando encuentres un error, no te preocupes. Todos los programadores del mundo encuentran bugs. A buscar y arreglar bugs se le llama **debugging** (depurar).

### ¿Qué es la consola o terminal?

Es una ventana donde el ordenador muestra mensajes y tú puedes escribir comandos. En Python, veremos los resultados de nuestros programas aquí.
